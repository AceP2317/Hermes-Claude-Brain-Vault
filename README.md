# Hermes — Living Architecture

A self-contained, **drillable** diagram of **Hermes**: a personal AI "second brain" built on
one Claude Code orchestrator running on a single cloud VPS, an Obsidian-vault knowledge layer
served over MCP, every Claude surface (Code, Desktop, claude.ai, web chat) reaching it through
identity-gated doors, and a three-tier privacy firewall.

### ▶ Live: https://acep2317.github.io/Hermes-Claude-Brain-Vault/

Drill from **Context → Containers → Components → Flows**, or open the **Reference** tab for the
per-surface doors matrix and the full service inventory. Hover a node to trace its links; click
to pin its details and the deciding rationale.

The diagram is *living* — it's generated from the project's infrastructure-as-code, so the
ports, services, users, and schedules reflect the real running system. Identifying details
(host IP, domains, provider/region, business and employer names) are redacted for publication.
